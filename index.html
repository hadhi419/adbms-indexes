Indexing Formulas

bfr = ⌊BlockSize / RecordSize⌋
b = ⌈TotalRecords / bfr⌉
Ri = KeySize + PointerSize
bfri = ⌊BlockSize / Ri⌋
bi = ⌈IndexEntries / bfri⌉
Search Cost: log₂(bi) + 1 block access.

Query Execution Lifecycle

1. Scanner: Breaks SQL into tokens (SELECT, FROM).
2. Parser: Syntax check. Checks grammar rules.
3. Validation: Checks if tables and columns exist in catalog.
4. Query Tree: Converts SQL to Relational Algebra tree.
5. Optimizer: Decides the best plan (uses statistics/indexes).

Select (σ) Search Methods

S1Linear Search: Brute force. Slowest. O(b).
S2Binary Search: Only for ordered files. O(log b).
S3aPrimary Index: Equality search on key.
S4Range Search: >, < using primary index.
S10Pointer Intersection: Using multiple indexes for AND.
Selectivity: (Satisfying Records) / (Total Records). 0 to 1 range. Lower value = Better filter.

Join (⨝) Algorithms

J1Nested-Loop: Default. Every record vs Every record.
J2Index-Based: Loop outer, use index for inner.
J3Sort-Merge: Best for large, physically sorted data.
J4Hash Join: Partition into buckets + Probe. Fastest for equality.

Exam Shortcuts

External Sorting Cost:
2b + (2b * log_m(nR))
b = blocks, m = buffer size
Pipelining vs Materialization:
Pipelining: No disk writes, faster.
Materialization: Stores temp results on disk.
Distinction: Duplicates are removed in relational PROJECT (π), but in SQL only if DISTINCT is used.
