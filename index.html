Indexing Structures

The Visual Cheat Sheet for Database Management Systems

Primary Index

Built on a unique, ordering key. Data is physically sorted. Usually sparse (one entry per block).

Sparse Index

Key: 101
Key: 104
Key: 107

Sorted Data File

B1 (101, 102, 103)
B2 (104, 105, 106)
B3 (107, 108, 109)
Quick Logic: Search for ID 102? Check index: 101 ≤ 102 < 104. Follow 101 pointer to Block B1 and scan.

Clustering Index

Built on non-unique ordering fields (e.g., DeptCode). Data is sorted by groups. Sparse index with one entry per distinct value.

Distinct Keys

IT
HR
Sales

Grouped Blocks

B1: IT, IT
B2: IT, HR
B3: HR, Sales
Tip: Pointers go to the FIRST block containing that value. Scan until the value changes.

Secondary Index

Non-ordering fields. Data is physically unsorted. Usually dense (one entry per record).

Dense Index

Email1
Email2
Email3

Unsorted Data

B2 (Email2)
B1 (Email4)
B3 (Email1)
B1 (Email3)

Math Foundation

Blocking Factor (bfr)
bfr = ⌊Block Size / Record Size⌋
Data Blocks (b)
b = ⌈Total Records / bfr⌉
Index Blocks (bi)
bi = ⌈Index Entries / bfri⌉
Search Accesses
log₂(bi) + 1 (data block)

Exam Memory Tips

  • Sparse Anchor records only (1 entry per block)
  • Dense Every record exists in the index
  • Clustering Sorted but non-unique key (entry per distinct value)
  • Secondary Key is NOT the physical sorting field
  • Efficiency Multilevel index reduces index search time
